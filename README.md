# Streamline IPTV

Production-grade Next.js App Router website for a legal IPTV platform, built for SEO, performance, and content scalability.

## Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- Metadata API and JSON-LD
- Zod-validated content models
- CMS-ready structured content architecture

## Setup

1. Install dependencies with `npm install`.
2. Create `.env.local` with `NEXT_PUBLIC_SITE_URL=http://localhost:3000`.
3. Start development with `npm run dev`.
4. Run type checks with `npm run typecheck`.
5. Run linting with `npm run lint`.
6. Build for production with `npm run build`.

## Vercel deployment

1. Import the repository into Vercel as a Next.js project.
2. Set `NEXT_PUBLIC_SITE_URL` to your production domain, for example `https://www.streamlineiptv.com`.
3. Use the default build command: `npm run build`.
4. Use the default output directory generated by Next.js.
5. Keep Node.js 20+ in the project settings.

The app also falls back to Vercel system URLs for preview and production environments, and preview deployments are automatically marked `noindex`.

## SEO strategy

- Server-rendered commercial and editorial pages with unique metadata for each route.
- Keyword coverage is split by search intent: broad IPTV, player, subscription, VOD, devices, features, and blog topics.
- Structured data is applied by page type: organization, website, breadcrumb, FAQ, service, and article.
- Internal linking routes users from broad intent pages into more specific supporting pages instead of duplicating the same copy.
- Blog and category pages are intentionally limited to useful, original content to avoid thin programmatic SEO patterns.
